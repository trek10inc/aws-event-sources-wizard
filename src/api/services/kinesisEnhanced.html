Amazon Kinesis with Enhanced Fanout is one of the only event sources that integrates directly with AWS Lambda that provides strict ordering guaruntees. It does this in a scalable fashion via sharding and the usage of partition keys to gauruntee ordering of all related records on that partition key.

The key difference between Amazon Kinesis and Amazon Kinesis with Enhanced Fanout is that the enhanced fanout streams allow you to create additional "replicated" streams so each consumer can get a dedicated pipe to consume with higher performance and the same ordering gauruntees. There is a soft limit of 5 subscribers via enhanced fanouts, which you can open a support ticket to get increased.

Key Info & Limits

  • 1MB / second / shard
  • 1000 records / second / shard
  • Soft limit of 5 enhanced fanout consumers
  • 24 hour data retention (optionally pay more for up to 7 days)
  • Order is only gaurunteed per partition key
  • Shards do not auto-scale
  • Number of shards can only be 3 times per 24 hours
  • Number of shards can only be increased by double or decreased by half.
  • The AWS Lambda integration polls each shard once per second
