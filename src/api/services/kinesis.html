Amazon Kinesis is one of the only event sources that integrates directly with AWS Lambda that provides strict ordering guaruntees. It does this in a scalable fashion via sharding and the usage of partition keys to gauruntee ordering of all related records on that partition key.

Key Info & Limits

  • 1MB / second / shard
  • 1000 records / second / shard
  • 24 hour data retention (optionally pay more for up to 7 days)
  • No more than 5 subscribers should be used
  • Order is only gaurunteed per partition key
  • Shards do not auto-scale
  • Number of shards can only be 3 times per 24 hours
  • Number of shards can only be increased by double or decreased by half.
  • The AWS Lambda integration polls each shard once per second
